API

This service has an API that can be interacted with from the /api route. All endpoints, with the exception of /api/just-ip, can have an IP preceding them. Before we talk about the dynamic API, we will talk about the static API. The difference is one (the dynamic API) specifies an IP for the backend to use, while the other (the static API) just uses your IP that you use to connect to the server with.

DISCLAIMER: Obviously the results of the frontend and the API will be skewed if you are connecting through a proxy server or VPN. These will mask your real IP with another. For more accurate results, disable these services.

Static API

/api/pretty (link)

The "pretty" API endpoint is the main endpoint that you get if you run cURL on this site. The server will redirect all default cURL requests to this endpoint for maximum usability.

GET arguments:
  • emoji
    • Type: boolean
    • Default: true
/api/just-ip (link)

The "just-ip" API endpoint just returns the IP that the server has received in the request.
